# Introduction

## What is Keq?

Keq is an HTTP client that can run in both browsers and Node.js. You can arm `Keq` to the teeth by adding `Middleware`!

[koa]: https://github.com/koajs/koa
[superagent]: https://github.com/ladjs/superagent

:::tip
`Keq` is inspired by [koa][koa] and [superagent][superagent].
It combines the middleware model from [koa][koa] with the chainable API design from [superagent][superagent].
:::

## Features

- Runs in both browsers and Node.js
- Supports chainable API
- Complete TypeScript support
- **[Convert Swagger to code](./libraries/keq-cli)**
- Extend functionality by adding `Middleware`. Official middleware includes:
  - [Timeout control](./features/timeout)
  - [Request retry](./features/retry)
  - [Concurrency control](./features/flow-control)
  - [Automatic response serialization](./features/serialize)
  - [Error handling](./libraries/keq-exception)
  - [Caching](https://github.com/keq-request/keq-cache)
